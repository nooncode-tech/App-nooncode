### Scenario 4 re-run note (path used: backfill, not new session)

This is a valid PASS for Scenario 4. F-V08's create-new-session-from-scratch path was already validated 2026-05-16 against localhost (PR #47 closure browser validation 4/4 PASS). What this smoke proves is that prod F-V08 backfill path works against a live Stripe session.

### Observations registered for tracking

1. **`paid_at` y `handoff_ready_at` usan `session.created` en vez del timestamp real del pago.** En `app/api/webhooks/stripe/route.ts:89-92` el handler hace `paidAt = new Date(session.created * 1000).toISOString()`. Esto refleja cuándo se creó la sesión Stripe (potencialmente días antes del pago), no cuándo el cliente efectivamente pagó. Para análisis de tiempo de cobro o reportes de flujo de caja real, esos campos son misleading. Magnitud: low — los timestamps con valor real (`confirmed_at` en seller_fees, `processed_at` en stripe_webhook_events, `created_at` en wallet_ledger_entries) están todos correctos. Recomendación: cambiar a `session.completed_at ?? now()` en un fix follow-up.

2. **Seller acreditado $100 por venta de $1.** El handler aplica `seller_fee_amount` literal sin validar que `seller_fee_amount <= activationAmount`. En operación real con seller_fee=100 y un cliente que pagara $50, el seller sería acreditado $100 (más de lo cobrado). Magnitud: medium — solo se manifiesta cuando seller elige un fee mayor al monto, pero la UI no lo previene. Tracked separately como TBD finding; no bloquea B1.3a.

3. **F-V08 backfill path = silent success on revisit.** Cuando un operador click "Crear link nuevo" sobre una proposal con session Stripe abierta, no se crea nada nuevo; solo se refresca la URL+expiry. La UI no le da feedback explícito de "estamos reusando la sesión existente". Magnitud: low UX — registrado como follow-up cosmético.

4. **G11 root cause: Production Branch misconfig + env var preview-branch lock.** Documentado en `docs/runbooks/cutover-pilot.md` §5.3 (G11 stale deploy). El runbook se actualizará en cierre B1.4 con esta nueva diagnosis verbatim para que la próxima vez sea 5 min en vez de 30.
